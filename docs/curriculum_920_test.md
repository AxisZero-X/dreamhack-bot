# 커리큘럼 920 테스트 문서

## 테스트 개요
- **테스트 목표**: 수강률 40% 달성
- **테스트 커리큘럼**: https://dreamhack.io/euser/curriculums/920
- **테스트 날짜**: 2026년 3월 6일
- **테스트 모드**: TEST_MODE=1 (빠른 테스트)

## 구현된 기능

### 1. 커리큘럼 URL 번호 프롬프트 입력
- `askCurriculumId()` 함수 추가
- 실행 시 커리큘럼 ID 입력 받음 (예: 920)
- 기본값: 916 (Enter 시)

### 2. ID/PW 프롬프트 입력
- `askCredentials()` 함수 추가
- 드림핵 이메일과 비밀번호 입력 받음
- 비밀번호 입력 시 마스킹(*) 처리

### 3. 자동 로그인 기능 개선
- `ensureLoggedIn()` 함수 개선
- 입력된 자격 증명으로 자동 로그인 시도
- **탭 키 문제 해결**: 이메일 입력 → Tab 키 → 비밀번호 입력 순서로 개선
- 비밀번호 필드 포커스 확인 및 폴백 로직 추가
- JavaScript 직접 실행으로 로그인 버튼 클릭 안정성 향상
- 실패 시 수동 로그인 대기 모드로 폴백

## 테스트 환경 설정

### .env 파일 설정
```bash
CHROME_PATH="/Applications/Google Chrome.app/Contents/MacOS/Google Chrome"
CURRICULUM_URL="https://dreamhack.io/euser/curriculums/920"
TEST_MODE=1
```

## 테스트 실행 방법

### 방법 1: 수동 입력
```bash
TEST_MODE=1 npm start
```
프롬프트에 순서대로 입력:
1. `920` (커리큘럼 ID)
2. `40` (목표 수강률)
3. 실제 드림핵 이메일
4. 실제 드림핵 비밀번호

### 방법 2: 자동 입력
```bash
echo -e '920\n40\nYOUR_EMAIL\nYOUR_PASSWORD' | TEST_MODE=1 node bot.js
```

## 문제 해결

### 자동 로그인 실패 시
1. 수동 로그인 대기 모드로 전환
2. 열린 Chrome 창에서 직접 로그인
3. 로그인 완료 후 자동 진행

### 테스트 모드 특징
- 딜레이 시간 최소화 (500ms ~ 1초)
- 빠른 테스트 가능
- 실제 실행 시 TEST_MODE=0 또는 제거 권장
