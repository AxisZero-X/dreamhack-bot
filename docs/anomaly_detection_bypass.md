# 드림핵 Anomaly Detection 우회 전략 문서

## 📋 개요

이 문서는 드림핵 서버의 anomaly detection 시스템을 우회하기 위해 구현된 전략을 설명합니다. 봇이 너무 빠른 속도로 강의를 수강하면 탐지될 수 있는 리스크를 분석하고, 이를 해결하기 위한 10가지 개선사항을 구현했습니다.

## 📊 리스크 분석 결과

### 현재 봇의 anomaly detection 탐지 위험도: 85% (매우 높음)

#### 주요 탐지 가능 패턴:
1. **퀴즈 풀이 속도**: 0.5~1초 (정상: 10~30초)
2. **강의 체류 시간**: 3~5초 (정상: 30초~2분)
3. **강의 간 전환**: 1~2초 (정상: 5~15초)
4. **퀴즈 정답률**: ~100% (정상: 60~80%)
5. **오답 패턴**: 순차적 브루트포스 (정상: 무작위)
6. **연속 수강**: 쉬지 않고 진행 (정상: 중간 휴식)
7. **일정한 패턴**: 모든 강의 동일 속도 (정상: 변동)

### Priority 1: 즉시 적용 필수

#### 1. 딜레이 대폭 증가 (`config.js`)

**변경사항:**

- 강의 간 전환: 1~2초 → 5~20초
- 퀴즈 재시도: 0.3~0.5초 → 3~8초

#### 2. 의도적 오답 추가 (`bot.js`)

#### 3. 무작위 휴식 시간 추가 (`bot.js`)
